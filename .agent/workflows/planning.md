# خطوات التخطيط لمهمة جديدة

## 1. فهم المتطلبات
- اقرأ طلب المستخدم بعناية
- حدد النطاق (scope) بوضوح
- اسأل عن أي شيء غير واضح

## 2. البحث في الكود
```bash
# البحث عن ملفات مرتبطة
grep -r "keyword" app/
# عرض هيكل المجلد
ls -la app/Http/Controllers/
```

## 3. تحديث Memory Bank
- افتح `.agent/memory-bank/activeContext.md`
- أضف المهمة الجديدة تحت "قيد التنفيذ"
- وثّق أي قرارات مهمة

## 4. إنشاء خطة التنفيذ
قم بإنشاء ملف في artifacts:
- `implementation_plan.md` للتغييرات الكبيرة
- حدد الملفات المتأثرة
- حدد خطوات الاختبار

## 5. مراجعة مع المستخدم
- اعرض الخطة للمراجعة قبل التنفيذ
- انتظر الموافقة

## 6. بعد الانتهاء
- حدّث `activeContext.md` بالمهام المكتملة
- حدّث `projectContext.md` إذا كان هناك تغييرات هيكلية
